# Use Case: show-meeting-cli

## Actor

Human Operator invoking the `ai-meeting show` CLI command from a terminal.

## Input

Positional:

| Arg | Type | Validation |
|-----|------|------------|
| `meetingId` | string | Required. A non-empty id produced by a prior `start_meeting`. Lookup against `FileMeetingStore`. |

Flags:

| Flag | Type | Validation |
|------|------|------------|
| `--format` | `text` \| `html` \| `json` \| `markdown` | Optional. Default `text`. |
| `--out` | path \| `-` | Optional. File path to write the rendered output. `-` means stdout (default for `text`). Required target kind: `html`/`markdown`/`json` default to stdout when `--out` is absent; explicit `--out path` writes to disk. |
| `--open` | boolean | Optional. Only meaningful with `--format html`. Writes to `${os.tmpdir()}/ai-meeting-<meetingId>.html` and spawns the OS default browser opener. Mutually exclusive with `--out <non-"-">`. |
| `--raw` | boolean | Optional. Expands the feed to include every persisted event type (`round.started`, `round.completed`, `job.started/completed/failed/cancelled`, `participant.dropped`, `meeting.ended`) in chronological order, not just `speech`/`pass`/`system` messages. |
| `--no-color` | boolean | Optional. Disables ANSI colors in `text` format. Implicit when stdout is not a TTY. |
| `--home` | absolute path | Optional. Override for `$AI_MEETING_HOME`. |

## Output

All formats produce a **header** block followed by a **body** block.

**Header (every format):**

- Meeting title, id.
- Status (`active` / `ended`), `createdAt`, `endedAt` (if any).
- Participants list: `id`, role (`facilitator` / `member`), `adapter` (`null` for facilitator), `status` (`active` / `dropped`).
- Jobs summary: for each Job — `id`, `status`, `terminationReason` (or `cancelReason` / `error.code`), `rounds` (number of Rounds executed; from `Job.rounds` — not `lastSeq`), `lastSeq` (raw event-log position, for debugging), `createdAt`, `finishedAt`.

**Body (without `--raw`):** messages in `seq` order, grouped by `round`. Rendered forms:

- `speech` — the author's text, prefixed/wrapped by format-specific decoration.
- `pass` — a compact pill / inline marker. The raw `<PASS/>` token is never printed verbatim.
- `system` — a separator with the incident text (drop reason, cancellation reason).

**Body (with `--raw`):** every event in `seq` order, type included (`round.started`, etc.), no grouping by round.

### Format: `text`

- Plain Unicode, TTY-friendly. ANSI colors on TTY unless `--no-color` / `NO_COLOR`.
- Header rendered as right-aligned key/value lines.
- Round boundaries rendered as `── Round N ──` rules; body rendered as `[r<N> <author> <kind>] <text>` with 2-space hanging indent for continuation lines.
- System messages rendered as `⚠ system: <text>` on its own line, indented to the round.
- Default destination: stdout.

### Format: `html`

- A single self-contained HTML5 document. Size target: < 300 KB for a typical 20-round meeting.
- All CSS inlined in a `<style>` block. No `<script>` elements. No `<link rel="stylesheet">`. No `<img src>` to remote URLs. No web fonts.
- Structure:
  - `<title>AI Meeting · <escaped title></title>`
  - A header card: meeting title, colored status pill (green `active`, grey `ended`), created/ended timestamps, participants list (each with an inline swatch of their assigned color).
  - A job card per Job, showing the termination reason with a matching color chip.
  - Rounds as collapsible `<details>` elements, default open for the last 3 rounds and collapsed for earlier ones.
  - `speech` — chat-bubble div aligned left/right alternately by participant index, tinted with the participant's color (HSL derived from a SHA-1 of `participantId`, with lightness clamped to 80–92 for legibility on white). The bubble's body is rendered as **a small Markdown subset** (see *Markdown rendering* below). The facilitator's opening Message is rendered as a centered, full-width bubble (visually distinct from member bubbles); its body uses the same Markdown pipeline.
  - `pass` — a small grey pill `passed` at the participant's column. Body is plain text (the literal `<PASS/>` token is never printed).
  - `system` — a centered divider with a `⚠` glyph and the incident text. Body is plain text — Markdown is NOT applied to system events.
  - Footer: generator signature `Generated by ai-meeting <version> · <ISO-8601 generation timestamp>`.

#### Markdown rendering (HTML format only)

Speech bubbles in the HTML report render their text as Markdown. The transformation pipeline is **escape-then-transform**: every character of the source text is HTML-escaped first, then a small in-tree Markdown converter introduces the supported tags. Raw HTML in message text is therefore impossible — the converter only adds tags it produces itself.

Supported subset (typical agent output):

| Markdown source | Rendered |
|---|---|
| `**bold**` or `__bold__` | `<strong>…</strong>` |
| `*italic*` or `_italic_` | `<em>…</em>` |
| `` `inline code` `` | `<code>…</code>` |
| ```` ```lang\n…\n``` ```` (fenced code block; fences may have 0–3 leading spaces) | `<pre><code class="lang-…">…</code></pre>` |
| `# Heading` … `### Heading` (h1–h3) | `<h1>` … `<h3>` |
| `- item` / `* item` / `+ item` (bullet list) | `<ul><li>…</li>…</ul>` |
| `1. item` (ordered list) | `<ol><li>…</li>…</ol>` |
| `> quoted line` | `<blockquote>…</blockquote>` |
| `---` / `___` on its own line | `<hr>` |
| `[label](url)` | `<a href="url">label</a>` if `url` matches `^(https?:\|mailto:)` — otherwise rendered as the original literal `[label](url)` |
| GFM table — header row `\| col \| col \|` + separator row `\|---\|---\|` + data rows `\| cell \| cell \|` | `<table><thead><tr><th>…</th></tr></thead><tbody><tr><td>…</td></tr></tbody></table>` |
| Blank line | Paragraph break |

Tables follow GFM conventions: a header row beginning and ending with `|`, immediately followed by a separator row whose cells contain only dashes (≥ 3 per cell, optionally with `:` for alignment that the renderer ignores), then any number of data rows in the same shape. A `|`-leading line without a separator below it is treated as paragraph text. Cell content goes through the inline pipeline, so bold, code spans, links, and allowlisted HTML inside cells all work.

Anything outside this subset (e.g. setext headings, reference-style links, definition lists) is rendered verbatim as escaped text. The `text`, `markdown`, and `json` formats ship the raw message text unchanged; only `html` performs this conversion.

**Inline HTML allowlist.** Agents commonly mix a handful of HTML tags into their replies (`<br>` for soft line breaks, `<b>` / `<i>` / `<u>` / `<kbd>` / `<sub>` / `<sup>` / `<mark>` for inline emphasis they could not express in pure Markdown). The renderer un-escapes a fixed allowlist of these tags **after** the escape-then-transform pipeline, so they appear as live HTML in the bubble. Rules:

- Allowed inline tags (open + close, no attributes): `b`, `strong`, `i`, `em`, `u`, `s`, `del`, `ins`, `code`, `sub`, `sup`, `kbd`, `mark`, `small`, `abbr`.
- Allowed void tags (self-closing, no attributes): `br`, `hr`. Both `<br>` and `<br/>` / `<br />` are accepted.
- **Attributes are forbidden.** `<b style="…">`, `<a href="…">`, `<img src="…">`, `<br data-x="y">` all stay escaped — only the bare tag forms are un-escaped. (Markdown-syntax links — `[label](url)` — remain the only way to introduce `<a href>` because the URL is sanitised.)
- The un-escape never touches content inside `<code>` or `<pre>`. So `` `<br>` `` in inline code stays as the literal text `<br>`, and a fenced code block preserves every angle bracket verbatim.
- Tags outside the allowlist (e.g. `<script>`, `<iframe>`, `<area>`, `<feature>`, custom names) stay escaped.

This keeps the document XSS-safe while letting agent output use the small set of HTML tags that LLMs reach for naturally.

Other rendering rules:

- The bubble container uses `white-space: normal`; block-level Markdown elements (`<p>`, `<pre>`, `<ul>`, etc.) handle their own whitespace. System dividers continue to use `pre-wrap` to preserve incident-text formatting.
- Code blocks are not syntax-highlighted (no JS allowed in the document). The optional language tag is preserved as a `lang-…` class for downstream re-rendering.
- Fenced code blocks accept up to three leading spaces before both fences; when present, up to that many spaces are stripped from each body line before escaping.
- The converter MUST sanitise link URLs: only `http://`, `https://`, and `mailto:` schemes produce an `<a>` tag. Any other scheme (`javascript:`, `data:`, `file:`, etc.) is rejected and the link is rendered as escaped literal text.
- Default destination: stdout. With `--out path` → file. With `--open` → tmp path + browser.

### Format: `markdown`

- GitHub-Flavored Markdown.
- Header as a fenced code block with `yaml` for metadata key/value pairs.
- Each round as `### Round N` heading.
- `speech` as a blockquote, with author attribution line above: `**<author>** — <ISO-8601 createdAt>`.
- `pass` as an italic line: `_<author> passed._`
- `system` as a horizontal rule followed by `> ⚠ <text>`.
- Default destination: stdout.

### Format: `json`

- A single JSON object: `{ meeting, participants, jobs, messages, generatedAt }`.
- `messages` — every `speech`/`pass`/`system` message when `--raw` absent; every event otherwise.
- Stable key order within each object (lexicographic within domain entities, or the order declared by the domain type).
- Pretty-printed with 2-space indentation.
- Default destination: stdout.

Exit code `0` on success.

## Flow

1. Parse argv. Reject unknown flags / missing `meetingId` with exit code `64` on stderr.
2. Reject `--open` without `--format html` with exit code `64`.
3. Reject `--open` combined with `--out <non-"-">` with exit code `64`.
4. Call `loadConfig()` to resolve `$AI_MEETING_HOME` (override honoured).
5. Instantiate `FileMeetingStore` pointed at that root.
6. Call `MeetingStorePort.loadMeeting(meetingId)`.
   - 6a. `MeetingNotFound` → stderr `meeting <id> not found`, exit code `3`.
7. Collect the body:
   - 7a. `--raw` absent: page through `MeetingStorePort.readMessagesSince({ cursor: undefined, limit: 500 })` in a loop until `hasMore === false`; concatenate into a single ordered array of Messages.
   - 7b. `--raw` present: call `MeetingStorePort.readAllEvents(meetingId)` and use the full event stream verbatim.
8. Select the renderer by `--format`. Pass it `{ meeting, participants, jobs, messages|events, raw, generatedAt }`.
9. Resolve destination:
   - 9a. `--open` → `path = ${os.tmpdir()}/ai-meeting-${meetingId}.html`; write file; spawn opener (see Rules).
   - 9b. `--out <path>` (not `-`) → write file atomically (tmp + rename).
   - 9c. `--out -` or absent → write to stdout.
10. Exit `0`. On any write failure during step 9, exit `2` with a stderr message.

## Errors

| Error | When | Exit code | Stream |
|-------|------|-----------|--------|
| `UsageError` | Missing `meetingId`, unknown flag, contradictory flag combo, invalid `--format` value. | `64` | stderr |
| `MeetingNotFound` | Step 6a. | `3` | stderr |
| `StoreUnavailable` | `$AI_MEETING_HOME` missing / unreadable / corrupt event log. | `2` | stderr |
| `WriteFailed` | Output file cannot be written (permissions, disk full). | `2` | stderr |
| `OpenerUnavailable` | `--open` requested but no platform opener found (`open`/`xdg-open`/`start`). | `0` (warn only) | stderr warning; file still written. |
| `InternalError` | Any other unhandled exception. | `1` | stderr |

## Side Effects

- Reads `$AI_MEETING_HOME/meetings/<meetingId>/events.jsonl` and the meeting's manifest.
- Writes the rendered output either to stdout or to the configured file path. The on-disk target is the only filesystem side effect.
- With `--open`, spawns a child process: `open <path>` (macOS), `xdg-open <path>` (Linux), `start "" <path>` (Windows). The child is detached and not awaited; the CLI returns as soon as it is spawned.

## Rules

- **Read-only against the store.** No mutation of events, manifest, or lockfile. Safe to run while `ai-meeting-server` is appending to the same meeting.
- **HTML escaping is mandatory.** All strings originating from messages, participant ids, titles, error text MUST pass through an HTML escaper before interpolation into the document. The renderer MUST NOT use string concatenation for attribute values without quoting; attribute values use double-quote delimiters and escape `"`, `&`, `<`, `>`, and `'` consistently.
- **No remote references in HTML.** The output document loads zero resources over the network. This is a property the HTML renderer unit test MUST assert on (e.g. grep for `src="http`, `href="http`, `<script` with a non-empty `src`).
- **Deterministic participant colors.** HSL computed as `hue = sha1(participantId)[0..3] % 360`, `saturation = 60%`, `lightness = 86%`. Pure function of id — two runs of `show` produce identical CSS. Facilitator always uses a neutral color (`#ededed`), regardless of id.
- **Message ordering.** Body is strictly ascending by `seq`. `system` messages appended by `handle-agent-failure` appear inside the round they belong to because their `round` field matches the running round number.
- **Grouping.** Without `--raw`, the body groups by `round`. Round 0 contains only the facilitator's opening message; rounds 1..N contain member responses and any system drop events. With `--raw`, no grouping is imposed — events are presented chronologically with their `type` included.
- **`--open` portability.** When `process.platform` is `darwin` → `open`; `linux` / `freebsd` → `xdg-open`; `win32` → `start "" <path>` via `cmd.exe /c`. On unknown platforms or when the opener binary is not on `PATH`, emit the `OpenerUnavailable` warning and still honour the file write.
- **Atomic file writes.** `--out path` writes `<path>.tmp` first, then `rename()` to `<path>`. Readers never see a half-written file.
- **No secrets.** `Participant.env` and any `Profile.env` MUST NOT be rendered in any format. `systemPrompt` IS rendered (it is the instruction, not a credential) but is visually de-emphasised in `html` / `text`.
- **Stable `generatedAt`.** The generator timestamp is derived from `Clock.now()` via `SystemClock`. In tests, a `FakeClock` allows deterministic snapshots.
- **`--raw` is a debug aid.** It is not part of the stable contract; event payload shapes can evolve across versions. The `text`/`markdown`/`html` formats hide this volatility behind the message-level abstraction.
