# Feature: meeting

## Purpose

Expose the public MCP surface for creating a Meeting, driving it forward with Facilitator messages, reading the Transcript as it grows, and closing it. This feature owns the Meeting aggregate and all Orchestrator-facing operations.

## Domain Entities

### Meeting

| Field | Type | Rules |
|-------|------|-------|
| `id` | `MeetingId` (branded UUID v4) | Generated by `IdGen` at creation. Immutable. |
| `title` | `string` | 1–200 characters after trimming. |
| `status` | `active` \| `ended` | Transitions `active → ended` once, monotonically. |
| `createdAt` | `Instant` | Supplied by `Clock` at creation. Immutable. |
| `endedAt` | `Instant` \| `null` | Set when `status` becomes `ended`. |
| `participants` | `Participant[]` | Non-empty. Exactly one Participant has `role = facilitator`. |
| `defaultMaxRounds` | integer 1–`VECHE_MAX_ROUNDS_CAP` | Default for `send_message.max_rounds` when the caller omits it. |

### Participant

| Field | Type | Rules |
|-------|------|-------|
| `id` | `ParticipantId` (string, `^[a-zA-Z0-9][a-zA-Z0-9_.-]{0,63}$`) | Unique within the Meeting. |
| `role` | `facilitator` \| `member` | Fixed for the lifetime of the Meeting. |
| `participantKind` | `model` \| `human` | Model Participants use an Adapter. Human Participants do not use an Adapter or Session. |
| `discussionRole` | `{ name: string, description: string, weight: number }` | Role metadata included in model prompt preambles and final synthesis weighting. `weight` must be finite and greater than `0`. |
| `isHumanParticipationEnabled` | boolean | `true` for Human Participants at creation unless toggled off by `set_human_participation`; `false` for Model Participants. |
| `adapter` | `codex-cli` \| `claude-code-cli` \| `null` | Required for Model Members; `null` for the Facilitator and Human Members. |
| `profile` | `string` \| `null` | Name of the Profile referenced at creation, if any. |
| `systemPrompt` | `string` \| `null` | Resolved final system prompt after Profile + overrides. Facilitator has `null`. |
| `workdir` | absolute path \| `null` | Passed to the Adapter. Facilitator has `null`. |
| `model` | `string` \| `null` | Adapter-specific model identifier (e.g. `gpt-5-codex`, `claude-opus-4-7`). `null` means adapter default. |
| `extraFlags` | `string[]` | Adapter-specific safe flag allow-list (see [agent-integration](../agent-integration/agent-integration.md)). |
| `status` | `active` \| `dropped` | Dropped Members stop receiving Turns. See [handle-agent-failure](../committee-protocol/handle-agent-failure.usecase.md). |
| `droppedAt` | `Instant` \| `null` | Set when `status` becomes `dropped`. |
| `droppedReason` | `string` \| `null` | Human-readable reason recorded on drop. |

### Message

| Field | Type | Rules |
|-------|------|-------|
| `id` | `MessageId` (branded UUID v4) | Generated by `IdGen`. |
| `meetingId` | `MeetingId` | Back-reference. |
| `seq` | integer ≥ 0 | Monotonic within the Meeting; used as the raw Cursor. |
| `round` | integer ≥ 0 | 0 for the Facilitator's opening Message; 1..N for Members' Round responses. |
| `author` | `ParticipantId` \| `"system"` | `system` for drop or termination markers. |
| `kind` | `speech` \| `pass` \| `system` | `speech` carries `text`; `pass` carries the literal `<PASS/>`; `system` carries an incident record. |
| `text` | `string` | Raw model output for `speech`/`pass`; structured incident text for `system`. |
| `createdAt` | `Instant` | Supplied by `Clock` at append time. |

### Round

Ephemeral aggregate used inside committee-protocol; not stored separately from Messages. See [committee-protocol](../committee-protocol/committee-protocol.md).

### Job

| Field | Type | Rules |
|-------|------|-------|
| `id` | `JobId` (branded UUID v4) | Generated by `IdGen`. |
| `meetingId` | `MeetingId` | Back-reference. |
| `status` | `queued` \| `running` \| `waiting_for_human` \| `completed` \| `failed` \| `cancelled` | See state transitions in [send-message](./send-message.usecase.md). |
| `createdAt` | `Instant` | Supplied by `Clock`. |
| `startedAt` | `Instant` \| `null` | Set when `status` becomes `running`. |
| `finishedAt` | `Instant` \| `null` | Set when `status` is any terminal value. |
| `maxRounds` | integer 1–`VECHE_MAX_ROUNDS_CAP` | Effective cap for this Job. |
| `lastSeq` | integer ≥ -1 | Highest Message `seq` appended by this Job. `-1` before the Facilitator Message lands. Reflects log position, NOT discussion progress. |
| `rounds` | integer ≥ 0 | Number of Rounds executed by this Job. `0` while the Job is still in Round 0 (facilitator Message just appended); incremented at the end of each Round. Frozen on terminal status. |
| `error` | `{ code: string, message: string }` \| `null` | Set when `status = failed`. |
| `cancelReason` | `string` \| `null` | Set when `status = cancelled`. |

### Human Turn

Persisted as event-log control records plus a normal Transcript Message authored by the Human Participant on submission.

| Field | Type | Rules |
|-------|------|-------|
| `requestId` | string UUID | Unique per Human Turn request. First valid submission wins. |
| `round` | integer ≥ 1 | The model Round that just completed. |
| `participantId` | `ParticipantId` | Must identify an active Human Member. |
| `agreeTargets` | `ParticipantId[]` | Model Members whose latest Round response can be agreed with. |
| `strengths` | `[1, 2, 3]` | Allowed agreement strengths. |
| `action` | `agree` \| `skip` \| `steer` | Submission kind. `agree` requires target + strength; `skip` records `<PASS/>`; `steer` requires free text and forces another model Round when the max-Rounds cap still permits it. |

### Synthesis

Stored as a `synthesis.submitted` Event after a Job reaches a terminal status. It is rendered separately from the Transcript and is not a `Message`.

### Cursor

| Field | Type | Rules |
|-------|------|-------|
| `value` | opaque string | Encodes a Meeting `seq` number. Monotonic within a Meeting. Returned by every Transcript-delivering query and round-tripped back by the Orchestrator. |

## Ports

### MeetingStorePort

Documented in [persistence](../persistence/persistence.md). This feature uses the following methods:

| Method | Purpose |
|--------|---------|
| `createMeeting(meeting)` | Persist a new Meeting and its initial Participants. |
| `appendMessage(meetingId, message)` | Append a Message event; assigns `seq`. |
| `readMessagesSince(meetingId, cursor, limit)` | Return a page of Messages beyond a Cursor. |
| `loadMeeting(meetingId)` | Load the Meeting aggregate (metadata + Participants + Job index). |
| `listMeetings(filter, paging)` | Return a paginated summary of Meetings. |
| `createJob(job)` | Persist a new Job in `queued`. |
| `updateJob(jobId, patch)` | Apply a status or field change to a Job. |
| `loadJob(jobId)` | Load a single Job. |
| `appendSystemEvent(meetingId, type, payload, at)` | Append Round, Human Turn, Human Participation, Synthesis, Job, and Meeting control events. |
| `markParticipantDropped(meetingId, participantId, reason, at)` | Persist a `participant.dropped` Event. |
| `endMeeting(meetingId, at)` | Persist a `meeting.ended` Event and flip status. |

### AgentAdapterPort

Documented in [agent-integration](../agent-integration/agent-integration.md). Consumed by this feature only through `committee-protocol`; the `meeting` slice never calls it directly.

### ClockPort

| Method | Input | Output |
|--------|-------|--------|
| `now` | — | `Instant` |

### IdGenPort

| Method | Input | Output |
|--------|-------|--------|
| `newMeetingId` | — | `MeetingId` |
| `newMessageId` | — | `MessageId` |
| `newJobId` | — | `JobId` |
| `newParticipantSessionId` | — | `string` (UUID v4) |

## Use Cases

### MCP tools (Orchestrator Agent)

- [start-meeting](./start-meeting.usecase.md) — Create a Meeting with a fixed roster.
- [send-message](./send-message.usecase.md) — Add a Facilitator Message and launch a Job that runs the committee discussion.
- [get-response](./get-response.usecase.md) — Poll a Job's status and consume Transcript deltas since a Cursor.
- [list-meetings](./list-meetings.usecase.md) — Enumerate Meetings with filters.
- [get-transcript](./get-transcript.usecase.md) — Read Transcript deltas outside the Job-polling flow.
- [end-meeting](./end-meeting.usecase.md) — Close a Meeting and release Adapter resources.
- [cancel-job](./cancel-job.usecase.md) — Abort an in-flight Job and finalise the Transcript up to the interruption point.
- `submit_human_turn` — Submit `agree`, `skip`, or `steer` for a pending Human Turn request.
- `set_human_participation` — Enable or disable a Human Participant for future pauses; disabling during a pending request auto-skips it.
- `submit_synthesis` — Store the facilitator's final synthesis for a terminal Job.

### CLI commands (Human Operator)

Read-only inspection via the `veche` binary, decoupled from any running MCP server.

- [list-meetings-cli](./list-meetings-cli.usecase.md) — Enumerate Meetings as an aligned table or JSON.
- [show-meeting-cli](./show-meeting-cli.usecase.md) — Render a full Transcript as `text`, `html`, `markdown`, or `json`. HTML output is a self-contained single-file report with chat-bubble layout.

## Dependencies

- [committee-protocol](../committee-protocol/committee-protocol.md) — executes the Job after `send_message` hands it over.
- [agent-integration](../agent-integration/agent-integration.md) — resolves Profiles and owns the `AgentAdapterPort`.
- [persistence](../persistence/persistence.md) — `MeetingStorePort`.
