# Feature: agent-integration

## Purpose

Define the single abstraction that every external LLM integration conforms to (`AgentAdapterPort`), plus the Profile system that resolves Participant configuration at Meeting creation. Ship two concrete adapters for v1 — Codex CLI and Claude Code CLI — that both adhere to the same port.

## Domain Entities

### Session

Per-Participant, per-Meeting state owned by the Adapter. The domain treats Session as opaque: it sees `id`, `adapter`, `status`, and an opaque `providerRef` handed back for subsequent Turns.

| Field | Type | Rules |
|-------|------|-------|
| `id` | string (UUID v4) | Generated by `IdGen.newParticipantSessionId`. Stable for the lifetime of the Meeting Participant. |
| `adapter` | `codex-cli` \| `claude-code-cli` | — |
| `participantId` | `ParticipantId` | — |
| `meetingId` | `MeetingId` | — |
| `providerRef` | string \| null | Opaque identifier produced by the Adapter (Codex `thread_id`, or the UUID the Claude Code CLI was told to use via `--session-id` on Turn 1 and `--resume` on Turn 2+). `null` before the first successful Turn. For Claude Code this field mirrors `id` and is set at `openSession` time; for Codex it is captured from the provider's `thread.started` event on Turn 1. |
| `status` | `open` \| `closed` | Transitions `open → closed` once. Dropped Participants close their Session on drop. |
| `openedAt` | `Instant` | — |
| `closedAt` | `Instant` \| null | — |

### Turn

Input to `AgentAdapterPort.sendTurn`.

| Field | Type | Rules |
|-------|------|-------|
| `session` | `Session` | Must be `open`. |
| `prompt` | `string` | The Facilitator's Message for Round 1, or the round-prefix payload for Rounds ≥ 2 (see Flow rules below). |
| `transcriptPrefix` | `MessageView[]` | Messages from other Participants in rounds this Participant has not yet seen, in ascending `seq` order. Specifically, every `speech`/`pass`/`system` Message with `round >= lastRound[participant]` authored by anyone else. On Round 1 it is exactly `[facilitator]`; on Round N+1 it is every other Member's Message from Round N. Older context is retained by the adapter's provider session (Codex `thread_id`, Claude Code `--resume`) — the protocol relies on session continuity to keep prompts compact. Injected into the Adapter's prompt — see [dispatch-turn](./dispatch-turn.usecase.md). |
| `systemPrompt` | `string` \| null | Applied on the *first* Turn only. Subsequent Turns inherit Adapter-native session memory. |
| `workdir` | absolute path \| null | Forwarded to the Adapter if supported. |
| `roundNumber` | integer ≥ 1 | For logging and for constructing the prefix preamble. |
| `timeoutMs` | integer ≥ 1000 | Wall-clock budget for this Turn. Enforced by the Adapter. |

### MessageView

Compact view of a prior Message used to build `transcriptPrefix`.

| Field | Type | Rules |
|-------|------|-------|
| `authorId` | string | Participant id or `"facilitator"`. |
| `authorRole` | `facilitator` \| `member` \| `system` | — |
| `round` | integer ≥ 0 | — |
| `text` | string | Raw Message text. |

### TurnResult

Output of `AgentAdapterPort.sendTurn`.

| Field | Type | Rules |
|-------|------|-------|
| `kind` | `speech` \| `pass` \| `failure` | Exactly one. |
| `text` | `string` \| null | Non-null for `speech` and `failure`. For `pass`, the raw model output is discarded. |
| `error` | `{ code: string, message: string, retryable: boolean }` \| null | Non-null for `failure`. |
| `providerRef` | string \| null | Updated provider reference, if the Adapter produced a new one this Turn (Codex emits the `thread_id` on the first `thread.started` event). |
| `durationMs` | integer ≥ 0 | Wall-clock duration including subprocess launch. |

### Profile

Named configuration stored in `${VECHE_HOME}/config.json`. Resolved at `start_meeting` by merging the Profile record with the caller-supplied overrides.

| Field | Type | Rules |
|-------|------|-------|
| `name` | string (`^[a-zA-Z0-9][a-zA-Z0-9_.-]{0,63}$`) | Unique within the config file. |
| `adapter` | `codex-cli` \| `claude-code-cli` | Required. |
| `model` | string \| null | Adapter-native model id. `null` uses Adapter default. |
| `systemPrompt` | string \| null | Appended with the Pass Signal instructions at runtime (see [dispatch-turn](./dispatch-turn.usecase.md)). |
| `workdir` | absolute path \| null | — |
| `extraFlags` | `string[]` | Must be an allow-listed flag for the chosen adapter (see each adapter's use case). |
| `env` | `Record<string,string>` | Extra environment for the subprocess. Must not override managed keys (`CODEX_API_KEY`, `HOME`, `PATH`). |

**User config file shape:**

```
{
  "version": 1,
  "profiles": [
    { "name": "codex-default", "adapter": "codex-cli", ... },
    { "name": "claude-code-senior", "adapter": "claude-code-cli", ... }
  ]
}
```

## Ports

### AgentAdapterPort

| Method | Input | Output | Errors |
|--------|-------|--------|--------|
| `capabilities` | — | `{ adapter: 'codex-cli'\|'claude-code-cli', supportsWorkdir: boolean, supportsSystemPrompt: boolean }` | — |
| `openSession` | `{ meetingId, participantId, sessionId, systemPrompt, workdir, model, extraFlags, env }` | `Session` with `status=open` | `AdapterNotAvailable`, `AdapterConfigInvalid` |
| `sendTurn` | `Turn` | `TurnResult` | `AdapterTurnTimeout`, `AdapterInvocationError`, `AdapterParseError` |
| `closeSession` | `Session` | `Session` with `status=closed` | `AdapterNotAvailable` (non-fatal) |

Error classes are defined at the port level:

| Error | Meaning | `retryable` |
|-------|---------|-------------|
| `AdapterNotAvailable` | The Adapter binary was not found or the configured auth is missing at process start. | `false` |
| `AdapterConfigInvalid` | A Participant option is outside the Adapter's supported surface (e.g. disallowed `extraFlags`). | `false` |
| `AdapterTurnTimeout` | A single Turn exceeded `timeoutMs`. | `true` |
| `AdapterInvocationError` | The subprocess exited with a non-zero code. | `true` if the exit code is in the Adapter's retryable set (defined per adapter), otherwise `false`. |
| `AdapterParseError` | The subprocess produced output that could not be parsed into a Message. | `false` |

### ClockPort, IdGenPort

Re-used from [shared](../../containers/veche-server.md).

## Use Cases

- [dispatch-turn](./dispatch-turn.usecase.md) — Build the prompt, invoke the Adapter, classify the outcome. Generic across adapters.
- [codex-cli-adapter](./codex-cli-adapter.usecase.md) — Concrete contract with `codex exec`.
- [claude-code-cli-adapter](./claude-code-cli-adapter.usecase.md) — Concrete contract with `claude -p`, including the Recursion Guard.

## Dependencies

- External binary: `codex`, `claude`.
- External env: `CODEX_API_KEY` (Codex only, optional — the CLI also accepts `~/.codex/auth.json` from a prior `codex login`), `HOME` (both).
- Internal: [committee-protocol](../committee-protocol/committee-protocol.md) consumes this port.
